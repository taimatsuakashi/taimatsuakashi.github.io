# 開発ルール（初版）

今後コンテンツが増えても迷わないための最小ルールです。

## 1. ディレクトリ方針

- アプリ紹介は `pages/apps/` 配下
- ブログは `pages/blog/` 配下
- 実験・検証は `pages/labs/` 配下
- 既存サンプル群は `実験室 管理番号1` として扱う

## 2. 命名ルール

- HTMLファイル名は `kebab-case`
- 管理番号は `no1`, `no2` のように半角英数字
- 画像名は用途が分かる英小文字（例: `app-timer-home.png`）

## 3. ページ作成ルール

- 各ページで `<meta name="viewport">` を必ず設定
- 共通スタイルは `assets/style.css` を利用
- 新規ページ作成時は必ず一覧ページにリンクを追加

## 4. ブログ運用ルール

- 1記事1HTML
- ファイル名は `YYYY-MM-DD-タイトル.html`
- 記事追加時は `pages/blog/index.html` にリンク追記

## 5. 公開前チェック

- リンク切れチェック（トップから2クリック以内で到達可能か）
- モバイル表示確認
- 誤字脱字確認
- 不要なデバッグコード削除

## 6. 将来の拡張方針

- 日本語中心で運用
- 英語対応が必要になったら `en/` ディレクトリを追加して段階的に多言語化
- 記事数が増えた段階で静的サイトジェネレーター導入を再検討
