# GitHub Pages運用メモ

## 1. 公開方式の選び方

GitHub Pagesには大きく2方式あります。

- ユーザーサイト: リポジトリ名が `ユーザー名.github.io`
- プロジェクトサイト: 任意のリポジトリ名

このリポジトリ名は `taimatsuakashi.github.io` なので、現在はユーザーサイト方式です。

おすすめ:

- 今のままユーザーサイト方式を継続
- トップURL直下に公開されるため運用が単純

## 2. パス運用の注意

ユーザーサイトはルート(`/`)運用が基本です。

- 画像・CSS・リンクは壊れやすいので、ページ階層に応じた相対パスを統一する
- 例: `pages/blog/index.html` からCSSを読むなら `../../assets/style.css`

## 3. 公開時の注意点

- `main` ブランチ公開か、`gh-pages` ブランチ公開かを固定する
- 大文字小文字を混在させない（Windowsで気づきにくいリンク切れ対策）
- 404対策として、将来必要なら `404.html` を追加する
- GitHub Pagesはサーバーサイド処理なし（PHP/DBは不可）

## 4. 触らない方がよいもの

- 既存公開URLを急に変更しない
- ファイル名を頻繁に改名しない（外部リンク切れ防止）

## 5. 推奨ルール

- 新規ページは `kebab-case` ファイル名で作成
- 画像は `assets/images/`、スクリプトは `assets/js/` に集約
- 1ページ公開ごとにトップまたは一覧ページに導線を追加
